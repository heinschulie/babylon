# Research: apps/web

## Research Question

Comprehensive documentation of the `apps/web` directory — its structure, routes, dependencies, configuration, and how it integrates with the rest of the monorepo.

## Summary

`apps/web` is the main learner-facing SvelteKit 2 application in the Babylon monorepo. It is an isiXhosa language-learning app ("Thetha") that lets users manage a phrase library, run audio-recording practice sessions scored by AI and human verifiers, browse vocabulary flashcards, read theory content, and manage settings (notifications, billing, locale, skin). It uses Convex as its real-time backend, Better Auth for authentication, Paraglide JS for i18n (English + isiXhosa), and shadcn-svelte via the `@babylon/ui` package for components. It deploys to Railway using the Node adapter.

### Root Layout (`+layout.svelte`)

Sets up Convex client (`setupConvex`), Better Auth client (`createSvelteAuthClient`), renders the `Header` from `@babylon/ui`, syncs user preferences for locale and skin from Convex on load.

File: `apps/web/src/routes/+layout.svelte`

### Practice Flow (Home Page)

The `/` route is the core of the app. It has three states:

1. **Dashboard** (no active session) — shows streak, "Start Session" card, vocabulary link, recent sessions accordion with average AI scores. A floating action button (FAB) also triggers session start.
2. **Active session** — shuffles all user phrases, presents English prompts one at a time with fly transitions, records audio via MediaRecorder, uploads to Convex storage, creates attempts, fires AI pipeline (`api.aiPipeline.processAttempt`) as fire-and-forget, advances immediately. Queue modes: once / shuffle / repeat.
3. **Session review** — accordion of attempts with AI scores (Sound/Rhythm/Phrase), user audio playback, verifier audio playback, fire gif indicator for unseen verifier feedback.

File: `apps/web/src/routes/+page.svelte`

### Server Hooks

Three hooks in sequence:
1. **Security headers** — `x-content-type-options`, `referrer-policy`, `permissions-policy`, HSTS in production
2. **i18n** — Paraglide middleware, replaces `%lang%` in HTML
3. **Auth** — extracts BetterAuth token from cookies into `event.locals.token`

File: `apps/web/src/hooks.server.ts`

### Auth

- Server-side: minimal `betterAuth()` instance for token extraction only (`src/lib/server/auth.ts`). Uses `SITE_URL` and `BETTER_AUTH_SECRET` env vars. Trusted origins include localhost ports and verifier site.
- Client-side: uses shared `authClient` from `@babylon/shared/auth-client`
- API routes: catch-all handler at `/api/auth/[...all]`
- Auth guard: practice and library pages redirect to `/login` when `!$isAuthenticated`

### i18n

- Paraglide JS with cookie + baseLocale strategy
- Two locales: `en` (English), `xh` (isiXhosa)
- Message files: `apps/web/messages/en.json` (209 keys), `apps/web/messages/xh.json` (209 keys, ~20 `[TODO]` entries for recent features)
- Compiled output: `src/lib/paraglide/` (gitignored)
- Locale switch: settings page calls `api.preferences.upsert` + `setLocale()`
- Layout syncs Convex-stored locale preference on load

### Vocabulary Sets

Static data in `src/lib/vocabularySets.ts` — 15 categories (colors, numbers, animals, transport, body parts, clothing, food, drinks, fruits/vegetables, family, days of week, weather, household, emotions, places) totaling ~160 English-Xhosa word pairs. Each item has a `searchTerm` used to fetch Unsplash images in the flashcard view.

### Styling

- `src/app.css` imports a single shared stylesheet: `packages/shared/src/styles/recall.css`
- `app.html` includes inline script for dark mode and skin detection from localStorage, loads Bebas Neue (display) + Public Sans (body) from Google Fonts
- Two skins: `default` ("Recall") and `mono` ("Monochrome"), applied via `data-skin` attribute on `<html>`
- shadcn-svelte configured via `components.json` with slate base color

### Configuration

- **SvelteKit**: Node adapter, CSP directives (auto mode), env dir points to monorepo root (`../..`)
- **Vite**: Paraglide plugin, Tailwind CSS plugin, SvelteKit plugin, allows filesystem access to `convex/` dir
- **Vitest**: jsdom environment for web tests, edge-runtime for Convex tests, includes `../../convex/**` test files
- **Railway**: Railpack builder, `bun install` + `bun run build:web`, starts with `node apps/web/build/index.js`, healthcheck on `/`

### Static Assets

`static/` contains: PWA manifest (`manifest.json` — app name "Language Recall"), service worker (`sw.js`), icons (72px badge, 192px, 512px), Thetha logo (avif + png), fire.gif, robots.txt.

### Testing

One test file: `src/lib/stores/auth.test.ts` — unit tests for shared auth stores (session, isAuthenticated, isLoading, user) using vi.mock to stub the BetterAuth client.

## Architecture Documentation

**Pattern: Practice-first UX**. The root route `/` is the practice page, not a dashboard or landing page. Library is secondary navigation. This prioritizes daily micro-sessions.

**Pattern: Fire-and-forget AI processing**. Audio submissions create the attempt and upload immediately, then dispatch `api.aiPipeline.processAttempt` without awaiting. The UI advances to the next phrase instantly. Review scores appear reactively via Convex subscriptions.

**Pattern: Dual scoring**. Attempts carry both AI scores (sound, rhythm, phrase accuracy) and optional human verifier scores from the separate verifier app. The fire.gif icon indicates unseen verifier feedback.

**Pattern: Shared styles, not shared i18n**. The UI package (`@babylon/ui`) has no i18n of its own — all translated strings are passed as props from the consuming app. Styles come from `packages/shared/src/styles/recall.css`.

**Pattern: Cookie-only locale**. No URL prefixes for locale. Paraglide uses cookie strategy. Server hooks transform `%lang%` in `app.html`.

**Pattern: Skin theming**. Two visual themes selected via `data-skin` HTML attribute, persisted in localStorage and Convex preferences.
